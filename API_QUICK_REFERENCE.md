# API Quick Reference - For Junior Developers

Copy this template and modify for your own API. This is the simplest possible REST API.

## 1. Create Controller

**File:** `app/Http/Controllers/Api/YourResourceController.php`

```php
<?php
namespace App\Http\Controllers\Api;

use App\Models\YourModel;
use Illuminate\Http\Request;
use App\Http\Controllers\Controller;

class YourResourceController extends Controller
{
    // GET /api/resources
    public function index(Request $request)
    {
        $items = YourModel::paginate($request->get('per_page', 15));
        return response()->json(['success' => true, 'data' => $items]);
    }

    // POST /api/resources
    public function store(Request $request)
    {
        $validated = $request->validate([
            'name' => 'required|string|max:255',
            'email' => 'required|email|unique:your_table',
        ]);

        $item = YourModel::create($validated);
        return response()->json(['success' => true, 'data' => $item], 201);
    }

    // GET /api/resources/{id}
    public function show(YourModel $resource)
    {
        return response()->json(['success' => true, 'data' => $resource]);
    }

    // PUT /api/resources/{id}
    public function update(Request $request, YourModel $resource)
    {
        $validated = $request->validate([
            'name' => 'sometimes|string|max:255',
            'email' => 'sometimes|email|unique:your_table,email,' . $resource->id,
        ]);

        $resource->update($validated);
        return response()->json(['success' => true, 'data' => $resource]);
    }

    // DELETE /api/resources/{id}
    public function destroy(YourModel $resource)
    {
        $resource->delete();
        return response()->json(['success' => true]);
    }
}
```

## 2. Add Route

**File:** `routes/api.php`

```php
Route::apiResource('resources', YourResourceController::class);
```

## 3. Create Test

**File:** `tests/Feature/YourResourceTest.php`

```php
<?php
namespace Tests\Feature;

use App\Models\YourModel;
use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;

class YourResourceTest extends TestCase
{
    use RefreshDatabase;

    public function test_get_all_resources()
    {
        YourModel::factory(5)->create();
        $response = $this->getJson('/api/resources');
        $response->assertStatus(200)->assertJson(['success' => true]);
    }

    public function test_create_resource()
    {
        $response = $this->postJson('/api/resources', [
            'name' => 'Test',
            'email' => 'test@example.com',
        ]);
        $response->assertStatus(201)->assertJson(['success' => true]);
    }

    public function test_get_resource()
    {
        $item = YourModel::factory()->create();
        $response = $this->getJson("/api/resources/{$item->id}");
        $response->assertStatus(200)->assertJson(['success' => true]);
    }

    public function test_update_resource()
    {
        $item = YourModel::factory()->create();
        $response = $this->putJson("/api/resources/{$item->id}", [
            'name' => 'Updated',
        ]);
        $response->assertStatus(200)->assertJson(['success' => true]);
    }

    public function test_delete_resource()
    {
        $item = YourModel::factory()->create();
        $response = $this->deleteJson("/api/resources/{$item->id}");
        $response->assertStatus(200)->assertJson(['success' => true]);
    }
}
```

## HTTP Status Codes

| Code | Meaning | When to Use |
|------|---------|------------|
| 200 | OK | GET, PUT successful |
| 201 | Created | POST successful |
| 204 | No Content | DELETE successful |
| 400 | Bad Request | Invalid request |
| 404 | Not Found | Resource doesn't exist |
| 422 | Unprocessable Entity | Validation failed |
| 500 | Server Error | Something broke |

## cURL Examples

```bash
# List
curl http://localhost:8080/api/resources

# Create
curl -X POST http://localhost:8080/api/resources \
  -H "Content-Type: application/json" \
  -d '{"name":"Test","email":"test@example.com"}'

# Show
curl http://localhost:8080/api/resources/1

# Update
curl -X PUT http://localhost:8080/api/resources/1 \
  -H "Content-Type: application/json" \
  -d '{"name":"Updated"}'

# Delete
curl -X DELETE http://localhost:8080/api/resources/1
```

## Validation Rules

```php
// Common rules
'name' => 'required|string|max:255',
'email' => 'required|email|unique:users',
'password' => 'required|string|min:8|confirmed',
'age' => 'sometimes|integer|between:1,120',
'url' => 'sometimes|url',
'date' => 'sometimes|date_format:Y-m-d',

// Unique rule (ignore own record)
'email' => 'unique:users,email,' . $user->id,

// Update rule (make optional)
'name' => 'sometimes|string|max:255',
```

## Response Format

All responses should follow:

```json
{
  "success": true,
  "message": "User created successfully",
  "data": {
    "id": 1,
    "name": "John",
    "email": "john@example.com",
    "created_at": "2026-02-05T10:00:00Z",
    "updated_at": "2026-02-05T10:00:00Z"
  }
}
```

Errors:
```json
{
  "message": "The given data was invalid.",
  "errors": {
    "email": ["The email has already been taken."]
  }
}
```

- [ ] All tests pass
- [ ] Documented endpoints (in comments or docs)

## Common Mistakes

❌ **Don't:** Return HTML on API  
✅ **Do:** Always return JSON

❌ **Don't:** Forget password hashing  
✅ **Do:** Use `bcrypt()` for passwords

❌ **Don't:** Return wrong status codes  
✅ **Do:** 201 for create, 200 for success, 422 for validation

❌ **Don't:** Skip validation  
✅ **Do:** Always validate input

❌ **Don't:** Forget to test  
✅ **Do:** Write tests for all endpoints
